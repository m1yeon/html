14_폼(form)태그

1. 폼 태그란?

- 사용자가 입력한 정보(아이디, 패스워드 등)을 서버로 전송하는 역할을 하며,
- 폼태그를 활용한 양식의 종류에는 로그인폼, 검색폼, 예약폼 등의 다양한 양식을 만들 수 있음.

2. 폼태그의 종류

form, input, select/option, textarea, label, button, fieldset, fieldset, legend

3. get / post

- 사용자가 버튼을 누르면 그 정보를 서버에 전송시킬 때, post=암호화 시킬지, get=시키지 않을지 선택하는 것 암호화 시키지 않으면 내가 입력한 내용이 홈페이지 주소에 뜸
- get : 암호화 하지 않으며, 테이터가 많지 않은 내용일 때 사용
- post : 암호화를 하며, 보낼 데이터가 많을 때 사용 (요즘엔 get 방식을 사용하지 않는 추세)
- 양식을 만들 때 하나로 묶어서 작업하지 않고 부분 부분 쪼개서 양식을 만들기 때문에 fieldset로 영역을 쪼개서 작업함

4. 폼태그 사용하기

폼그룹 제목 1. 텍스트박스 : 문자를 입력하기 위한 박스로서 '아이디', '검색', '주소', '문의하기 제목' 등

2. 패스워드 박스 : 비밀번호, 비밀번호확인, 주민번호, 카드번호, 개인정보 작성 시 글자가 보이지 않게 '기호'로 표시되게 함

3. 라디오버튼 : 여러개의 항목 중 하나만 선택이 가능한 양식 동의함 동의안함

남 여

다음 보기 중 맞는 항목을 선택하세요



4. 체크박스 : 체크박스(여러개의 항목들 중 사용하자 원하는 항목만 선택 할 수 있는 양식)
나의 취미

5. 선택목록 : 쇼핑몰 옵션선택, 전화번호 입력 통신사 선택, 예매목록, 인원수, 언어설정, 패밀리사이트 등의
양식을 만들고자 할 때 사용하는 양식

6. 다중텍스트 박스 : 한 줄 입력이 아닌 여러 줄의 내용을 입력하고자 할 때 사용

7. 파일첨부 : 게시판이나 이메일 양식에서 상대방에서 파일을 첨부하여 보내기 위한 양식


8. 버튼 : 전송하기, 취소하기, 일반버튼, 이미지버튼
